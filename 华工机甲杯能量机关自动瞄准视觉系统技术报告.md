# 华工机甲杯能量机关自动瞄准视觉系统技术报告
                                       作者:廖家强

## 1. 引言

在这次比赛中，为高效击打旋转的大小符靶标，我设计并实现了一套基于单目视觉的自动瞄准系统。系统通过 USB 相机实时捕获画面，识别转盘圆心、锁定激活颜色、跟踪并预测目标装甲板位置，最后通过串口向云台控制板发送角度偏差，完成闭环瞄准。

## 2. 系统总体设计

### 2.1 硬件组成
- **相机**：免驱 USB 工业相机（640×480@30fps，MJPEG 格式）
- **计算平台**：Windows 笔记本。
- **下位机**：STM32 云台控制板，通过串口 (115200bps) 接收角度指令，驱动舵机或电机。
- **结构**：眼在手，相机固定在发射云台上方，云台移动时相机一起转动。

### 2.2 软件架构
程序以 C++ 编写，使用 OpenCV 处理图像，Windows API 操作串口。主循环流程为：
1. 帧获取 → 2. 圆心检测 → 3. 中心颜色识别 → 4. 目标装甲板检测 → 5. 运动预测 → 6. 角度计算 → 7. 串口指令输出。
  同时支持键盘指令切换状态（开始瞄准、重置锁定）。

## 3. 视觉算法核心模块

### 3.1 图像预处理
相机直接输出 MJPEG，解码为 BGR 后转换为 HSV 色彩空间，便于颜色分割。针对赛场强光和激光干扰，采用**手动曝光**模式并将曝光补偿设为 -6，使背景大幅压暗，减少杂波干扰，同时保留目标色块的饱和信息。

### 3.2 圆心检测
转盘中央有一个固定的圆形图案，作为圆心基准。检测方法：
1. 灰度化 + 高斯模糊。
2. **霍夫圆检测**（`HoughCircles`），参数经调试：最小半径 8，最大半径 80，累加器阈值 100。
3. 从检测到的圆中选择**距离画面物理中心最近的一个**，作为转盘圆心（因为相机大致对准转盘，视场中央的圆就是靶盘圆心）。

为提高稳定性，设计了**鲁棒跟踪机制**：
- 维护上一帧的有效圆心位置 `last_valid_center`。
- 若当前帧的圆心与上一帧的距离小于 120 像素，则更新；否则弃用当前结果，沿用上一帧坐标。
- 若连续丢失超过 30 帧（约 1 秒），重置锁定，重新全局搜索。

### 3.3 中心颜色识别与锁定
比赛规则要求识别特定颜色，需识别转盘中心点的颜色以确定攻击目标。实现方法：
- 在圆心周围取 28×28 的方形 ROI，提取 HSV 直方图。
- 与预定义的颜色阈值表（红、紫、绿、黄等 8 种颜色）逐一比较，统计落入各颜色范围的像素数。
- 选择计数最多且超过 40 个像素（防止低曝光下的随机噪点误判）的颜色作为锁定颜色。
- 颜色锁定后，后续帧直接使用该颜色进行目标搜索，不再重复识别。

### 3.4 目标装甲板检测
目标装甲板是固定在转盘臂上的方形色块，与中心同色但面积小于中心圆盘（便于区分）。我的策略是：
1. 根据锁定颜色在整帧生成二值掩码，进行开运算（3×3）和闭运算（5×5）去除噪点并填充孔洞。
2. 查找所有外部轮廓，过滤面积（50~25000）和轮廓矩计算出质心。
3. 将所有候选色块按**面积降序排序**。由于中心圆盘面积远大于臂上的方形装甲板，**最大的色块就是我们要打击的目标**（即使同一颜色出现多个，也遵循此逻辑）。
4. 增加距离约束：目标质心到圆心的距离必须在 40~150 像素之间，过近（中心圆盘本身）或过远（轨道外干扰）的色块将被舍弃。
5. 采用连续跟踪辅助：若上一帧已跟踪到目标，当前帧优先选择距离上一帧质心最近的候选（且距离<40px），有效防止圆心跳到背景方灯上。

### 3.5 运动预测与开环盲推
能量机关的转速是变化的（正弦规律）。若仅使用当前帧位置计算误差，会因图像传输与处理延迟导致射击滞后。因此我们设计了**角速度估计 + 前馈预测**系统：
- 计算目标相对圆心的极坐标角度 `θ` 和极径 `R`。
- 通过帧间差分估算角速度 ω = Δθ/Δt，并用一阶低通滤波器（0.2×旧值 + 0.8×新值）平滑，兼顾响应速度和稳定性。对 ω 做 ±2.5 rad/s 的物理限幅。
- 引入 0.25s 的延迟补偿，预测目标角度 `θ_pred = θ + ω × 0.25`，结合固定极径 `R`（经 85% 历史权重平滑，抵抗质心抖动）反解出预测的直角坐标。
- **抗遮挡开环盲推**：当目标短暂丢失（如进入暗区或反光，小于 15 帧），系统利用最后已知的角速度与极径继续外推位置，并合成虚拟坐标发送给云台，避免“一丢就停”的抽风现象。超过 15 帧仍未找回则放弃，重新搜索。

### 3.6 云台控制指令生成
- 采用**角度误差法**：将目标预测点映射到相机归一化平面，计算其与画面中心（320,240）的偏角。
- 偏角计算：`ax = -atan2(dx, fx) * 180/π`，同理 ay。取负号是因为云台跟随需消除画面偏差（反向控制）。
- **去除传统死区**：由于目标始终在转动，若加入死区会导致云台追上后停转，瞬间又被甩开，产生“抽搐”。我们直接输出微小误差，由下位机速度环积分平滑。
- 数据包格式 `#X<水平10倍值>Y<垂直10倍值>!`，角度限制在 ±45° 内。
- 串口发送采用**非阻塞超时（10ms）**，防止下位机故障导致主线程卡死。

## 4. 关键问题分析与解决方案(运用AI的思路)

| 问题                                                 | 分析                                                 | 解决方案                                                     |
| ---------------------------------------------------- | ---------------------------------------------------- | ------------------------------------------------------------ |
| **串口写入阻塞导致帧率暴跌**                         | 默认 WriteFile 无限等待，下位机卡死会拖死整个程序。  | 配置 WriteTotalTimeoutConstant = 10ms，超时立即放弃发送。    |
| **低曝光下颜色误判**                                 | 背景随机噪点可能匹配到颜色表，导致错误锁定。         | 中心采样要求有效像素 > 40；只锁定检测到的、明确属于预定义的颜色。 |
| **同颜色多色块干扰（中心圆盘 vs 装甲板 vs 环境灯）** | 转盘上相同颜色的中心圆盘面积远大于目标，可能被误选。 | 按面积排序，取最大者为目标；同时距离圆心 40~150px 限制。     |
| **目标因高速旋转产生拖影或形变**                     | 形变导致质心跳动，角度计算不准。                     | 固定极径思想：物理半径不变，用历史平均半径钳制当前测量半径（85%旧值），强制平滑。 |
| **图像处理与串口传输延迟**                           | 100ms+ 的延迟让瞄准点落后于实际目标位置。            | 角速度前馈预测，超前 0.25s 打点。                            |
| **目标短暂丢失导致控制断流**                         | 遮挡或高反光区域可能使目标消失数帧。                 | 盲推系统：维持最近角速度和半径，外推至多 0.5 秒（15帧），实现无缝衔接。 |
| **霍夫圆检测不稳定**                                 | 光照变化可能让圆心跳动。                             | 结合帧间距离约束（120px），过滤异常值；超时重置机制。        |
| **在识别过程中方形色块识别点偶尔会突变**             | 因背景同色杂光导致的目标坐标瞬间跳跃                 | 角度突变检测超过45度时视为误检，进入盲推模式                 |

**AI的运用思路:**先将比赛的具体要求和规则以文件和图片形式发给豆包和deepseek等国内AI,要求其根据提供的资料总结出代码在比赛中需要实现的功能,后将总结发给chatgpt和gemini等,要求其对总结做出改善并生成相应的代码,后使用VS运行代码,并使用部署在VS的Github copilot在调试过程中根据实际问题进一步完善代码.



## 5. 工作流程与状态机

系统通过键盘 `S` 键启动瞄准（发送 #V! 给下位机），`R` 键重置颜色锁定，`ESC` 退出。整体运行状态：
1. **未锁定**：实时检测圆心，识别中心颜色，若成功锁定则进入跟踪状态。
2. **跟踪中**：持续跟踪目标装甲板，计算并输出角度指令。
3. **丢失恢复**：短时丢失启用盲推；长时丢失回到未锁定，重新搜索。
4. **退出**：发送 #Q! 通知下位机关闭激光，安全退出。

## 6. 编译与部署指南

### 6.1 环境依赖
- Windows 10/11
- Visual Studio 2019/2022 (MSVC)
- OpenCV 4.5+ (需包含 `opencv_world` 库)
- 无额外第三方库

### 6.2 编译步骤
1. 创建 C++ 控制台工程，配置 Release x64。
2. 设置包含目录：OpenCV 的 `include` 文件夹。
3. 设置库目录：OpenCV 的 `x64/vc16/lib`，添加 `opencv_world4xx.lib`。
4. 将提供的源码粘贴至主文件，编译生成 EXE。
5. 拷贝 OpenCV 的 `opencv_world4xx.dll` 到 EXE 同目录。

### 6.3 运行前配置
- **相机**：代码默认打开索引 1 的 USB 相机（`cap.open(1, CAP_DSHOW)`）。若需更改，修改此参数。程序会一直等待相机就绪。
- **串口**：默认 COM7，波特率 115200。确保设备管理器端口号一致，或修改 `initSerial("COM7")`。
- **颜色表**：根据赛场实际装甲板颜色在 `colorList` 向量中调整 HSV 阈值。可借助 OpenCV 滑块工具事先标定。
- **曝光**：若画面过亮或过暗，调整 `cap.set(CAP_PROP_EXPOSURE, -6)` 的值（范围通常 -13 到 -1）。

### 6.4 运行流程
1. 连接相机和串口线，启动程序。
2. 将相机对准能量机关转盘，待画面中心出现圆心圆点后，按 `S` 键开始瞄准。
3. 系统自动锁定颜色并跟踪目标，云台跟随。按下 `R` 可强制重新锁定颜色。
4. 需要停止时按 `ESC`，程序发送关闭指令并退出。

## 7. 总结
本项目针对能量机关高动态、强干扰的挑战，构建了一套稳定、低延迟的视觉瞄准方案。通过**多级鲁棒滤波、运动预测、开环盲推**等设计，有效抑制了丢帧、遮挡、颜色干扰等问题。
